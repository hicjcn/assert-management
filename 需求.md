# 资产管家需求文档

## 一、产品目标

开发一个简洁、好用的个人资产管理工具，用于记录各类资产账户、追踪资产变化、查看资产/负债/净资产图表，并根据收入和支出情况制定存钱目标。

产品核心目标：

- 清楚知道自己当前有多少钱、欠多少钱、净资产是多少。
- 能记录每个账户的资产变动历史。
- 能通过图表观察资产趋势、分类占比和账户排行。
- 能根据目标金额和每月收支预估达成时间。

## 二、MVP 范围

第一版优先实现以下能力：

- 资产账户新增、编辑、归档。
- 账户金额增加、减少、直接设定。
- 自动生成资产变更记录。
- 首页展示总资产、总负债、净资产和目标进度。
- 支持资产、负债、净资产的趋势图、分类占比图和排行榜。
- 支持设定存钱目标并计算预计达成时间。
- 支持查看、筛选所有变更记录。

后续增强功能放在文档末尾，不影响第一版开发。

## 三、核心概念

### 3.1 资产账户

资产账户是用户录入资产的基本单位，例如：

- 招商银行储蓄卡
- 微信零钱
- 支付宝余额
- 花呗
- 招行信用卡
- 基金账户
- 房贷账户

每个账户都属于一个分类，并拥有一个当前金额。

### 3.2 正资产与负资产

账户分为正资产账户和负资产账户。

- 正资产账户：现金、借记卡、虚拟账户、投资账户、债券账户等。
- 负资产账户：信用卡、负债账户等。

信用卡和负债账户录入的是负资产。统计净资产时需要从总资产中扣除。

### 3.3 净资产

净资产计算规则：

```text
净资产 = 正资产总额 - 负债总额
```

其中负债总额按绝对值展示，例如信用卡欠款 5000 元，负债总额展示为 5000 元。

## 四、账户分类

系统默认支持以下分类：

- 现金
- 借记卡
- 信用卡
- 虚拟账户
- 投资账户
- 负债账户
- 债券账户

分类说明：

| 分类 | 类型 | 示例 |
| --- | --- | --- |
| 现金 | 正资产 | 手头现金、备用金 |
| 借记卡 | 正资产 | 工资卡、储蓄卡 |
| 信用卡 | 负资产 | 招行信用卡、建行信用卡 |
| 虚拟账户 | 正资产 | 微信零钱、支付宝余额 |
| 投资账户 | 正资产 | 股票账户、基金账户、理财账户 |
| 负债账户 | 负资产 | 房贷、车贷、借款 |
| 债券账户 | 正资产 | 国债、债券基金、固定收益账户 |

后续可支持用户自定义分类。

## 五、资产录入与账户管理

### 5.1 新增账户

新增资产账户时需要填写：

- 账户名称，必填。
- 账户分类，必填。
- 当前金额，必填。
- 是否计入统计，默认计入。
- 备注，可选。

新增账户后，系统自动生成一条初始化变更记录。

### 5.2 编辑账户

支持修改：

- 账户名称
- 账户分类
- 是否计入统计
- 备注

编辑账户基本信息不会自动改变金额。

### 5.3 归档账户

支持归档不再使用的账户。

归档规则：

- 归档账户不在默认账户列表中展示。
- 归档账户默认不参与统计。
- 归档账户的历史变更记录仍然保留。

### 5.4 金额变更

账户金额支持三种变更方式：

| 变更方式 | 说明 | 示例 |
| --- | --- | --- |
| 增加 | 在当前金额基础上增加 | 工资到账 +10000 |
| 减少 | 在当前金额基础上减少 | 支出 -500 |
| 直接设定 | 直接把账户金额修正为指定值 | 基金账户更新为 23000 |

每次金额变更都必须生成一条变更记录。

### 5.5 金额输入规则

- 正资产账户金额通常为正数。
- 信用卡和负债账户金额按负债金额录入，统计时按负债处理。
- 为避免理解混乱，页面上负债账户可以显示为“欠款金额”，但底层统计按负资产计算。

## 六、变更记录

### 6.1 记录内容

每条变更记录包含：

- 所属账户
- 账户分类
- 变更类型：初始化、增加、减少、直接设定、修正
- 变更前金额
- 变更金额
- 变更后金额
- 备注
- 变更时间
- 创建时间

### 6.2 变更记录展示

支持展示所有资产账户的变更记录。

列表字段：

- 时间
- 账户名称
- 分类
- 变更类型
- 变更金额
- 变更后金额
- 备注

### 6.3 筛选与排序

变更记录支持：

- 按账户筛选
- 按分类筛选
- 按变更类型筛选
- 按时间范围筛选
- 按金额大小排序
- 按时间倒序展示，默认最新记录在前

### 6.4 修正与撤销

第一版可以先支持“修正”功能，不一定支持真正删除历史记录。

修正规则：

- 如果录错金额，可以新增一条修正记录。
- 修正记录需要保留原始记录，避免历史数据丢失。
- 后续可支持撤销最近一次变更。

## 七、首页仪表盘

首页作为主入口，展示资产总览。

### 7.1 核心指标

首页展示：

- 总资产
- 总负债
- 净资产
- 本月净资产变化
- 本月收入类增加金额
- 本月支出类减少金额
- 当前存钱目标进度
- 最近一次更新时间

### 7.2 快捷操作

首页提供快捷入口：

- 新增账户
- 记录变更
- 查看统计图表
- 查看目标计划
- 查看变更记录

### 7.3 空数据状态

用户首次进入时，如果没有账户数据，需要展示友好的空状态：

- 提示用户添加第一个资产账户。
- 提供“新增账户”按钮。
- 不展示空图表。

## 八、统计图表

统计图表分为资产图表、负债图表、净资产图表。

### 8.1 通用图表要求

三类图表都支持：

- 趋势图
- 分类占比图
- 排行榜
- 时间范围筛选
- 分类筛选
- 账户筛选

默认展示最近 12 个月数据。

时间范围支持：

- 最近 3 个月
- 最近 6 个月
- 最近 12 个月
- 全部
- 自定义时间范围

### 8.2 资产图表

资产图表只统计正资产账户。

需要包含：

- 资产走势图，使用折线图。
- 资产分类占比，使用饼图或环形图。
- 资产排行榜，按账户金额从高到低排序。

### 8.3 负债图表

负债图表只统计信用卡和负债账户。

展示规则：

- 负债金额按绝对值展示。
- 趋势图展示负债总额变化。
- 分类占比展示不同负债分类占总负债的比例。
- 排行榜展示欠款金额最高的账户。

### 8.4 净资产图表

净资产图表展示资产扣除负债后的结果。

需要包含：

- 净资产走势图，使用折线图。
- 净资产分类贡献占比。
- 净资产账户贡献排行榜。

### 8.5 月度统计口径

走势图按月统计时，采用以下规则：

- 每个月取该月最后一次账户金额作为月末金额。
- 如果某账户某个月没有变更，则沿用最近一次金额。
- 新增账户从创建月份开始参与统计。
- 不计入统计的账户不参与图表计算。

### 8.6 图表视觉风格

整体视觉要求：

- 使用马卡龙色系。
- 页面简洁、舒适、留白适中。
- 资产、负债、净资产使用不同主色。
- 同一个分类在不同图表中颜色保持一致。
- 图表需要有空状态展示。
- 移动端和桌面端都要适配。

建议配色方向：

| 类型 | 建议颜色 |
| --- | --- |
| 资产 | 柔和绿色、蓝绿色 |
| 负债 | 柔和红色、珊瑚色 |
| 净资产 | 柔和蓝色、紫蓝色 |
| 现金 | 淡黄色 |
| 借记卡 | 天蓝色 |
| 信用卡 | 珊瑚粉 |
| 虚拟账户 | 薄荷绿 |
| 投资账户 | 淡紫色 |
| 负债账户 | 浅红色 |
| 债券账户 | 浅橙色 |

## 九、目标计划

### 9.1 目标设定

支持设定一个或多个存钱目标。

目标字段：

- 目标名称
- 目标金额
- 关联资产账户，可选；支持关联一个或多个
- 预计每月工资收入
- 每月房租
- 每月餐费
- 每月生活费
- 每月其他花销
- 预计每月其他收入
- 预计一次性收入
- 预计一次性支出
- 备注

当前已完成金额不再手动维护：

- 关联了资产账户时，自动汇总所选未归档资产账户的当前金额。
- 未关联账户时，自动使用当前总净资产，口径与首页一致。
- 账户金额发生变化后，目标进度同步更新。

### 9.2 计算规则

每月可存金额：

```text
每月可存金额 = 每月总收入 - 每月总支出
```

距离目标还差：

```text
剩余目标金额 = 目标金额 - 自动计算的当前已完成金额
```

计划达成月数：

```text
计划达成月数 = 剩余目标金额 / 每月可存金额
```

如果每月可存金额小于或等于 0，需要提示用户当前收支无法达成目标。

趋势达成使用与当前已完成金额相同的账户范围，根据最近 6 个自然月的账户实际净增长计算：

```text
月均趋势增长 = 最近 6 个自然月实际净增长 / 可观察月数
趋势达成月数 = 剩余目标金额 / 月均趋势增长
```

- 账户不足 6 个月时，只按账户创建后的已有月份计算。
- 创建账户时登记的初始金额计入账户创建当月的增长；初始负债按净资产方向计为负增长。
- 总净资产模式下，负债增加计为负增长，负债减少计为正增长。
- 没有有效变更记录时显示“数据不足”；月均增长小于或等于 0 时显示“暂无增长趋势”。

### 9.3 目标展示

目标计划页面展示：

- 目标名称
- 目标金额
- 当前进度
- 完成百分比
- 每月可存金额
- 计划达成时间
- 趋势达成时间及近 6 个月月均增长
- 距离目标还差多少钱

### 9.4 目标增强

后续可以支持：

- 多目标优先级。
- 投资收益率假设。
- 年终奖、奖金等非固定收入。
- 按保守、中性、乐观三种方案估算达成时间。

## 十一、数据模型建议

### 11.1 账户表

账户字段：

- id
- name
- category
- account_type，asset 或 liability
- current_amount
- include_in_stats
- archived
- note
- created_at
- updated_at

### 11.2 变更记录表

变更记录字段：

- id
- account_id
- account_name_snapshot
- category_snapshot
- change_type
- before_amount
- change_amount
- after_amount
- note
- changed_at
- created_at

### 11.3 目标表

目标字段：

- id
- user_id
- name
- target_amount
- one_time_income
- one_time_expense
- note

目标与资产账户通过 `goal_accounts` 关联；没有关联记录表示使用总净资产。月度收支配置统一保存在 `settings`，目标当前已完成金额不重复落库。

## 十二、数据导入导出

第一版至少支持导出。

### 12.1 数据导出

支持导出：

- 账户列表
- 变更记录
- 目标计划

导出格式：

- CSV
- 后续可支持 Excel

### 12.2 数据导入

后续支持从 CSV 或 Excel 导入历史数据。

导入时需要处理：

- 账户名称匹配。
- 分类匹配。
- 金额格式校验。
- 重复数据识别。

## 十三、隐私与安全

资产数据较敏感，需要考虑隐私保护。

建议支持：

- 不记录完整银行卡号，只记录账户昵称。
- 本地数据存储优先。
- 可选访问密码或应用锁。
- 导出数据时提醒用户妥善保存。
- 后续可支持数据加密备份。

## 十四、交互细节

### 14.1 金额展示

- 金额统一保留两位小数。
- 大金额使用千分位展示。
- 负债页面展示为正数欠款，避免图表出现负数比例。
- 净资产允许为负数。

### 14.2 表单校验

- 账户名称不能为空。
- 金额必须是数字。
- 目标金额必须大于 0。
- 每月收入和支出不能为负数。
- 直接设定金额不能为空。

### 14.3 操作反馈

- 新增成功后提示。
- 修改成功后提示。
- 金额变更成功后提示，并展示变更后余额。
- 删除或归档前需要二次确认。

## 十五、后续增强

后续版本可以考虑：

- 自定义账户分类。
- 多币种资产管理。
- 预算管理。
- 收支流水管理。
- 周期性自动入账，例如工资、房租、贷款。
- 数据图表导出为图片。
- 数据备份与恢复。
- 云端同步。
- 移动端 PWA。
- 账户图标自定义。
- 资产健康度评分。
- 负债还款计划。
- 投资收益率统计。

## 十六、暂不做范围

第一版暂不考虑：

- 自动同步银行数据。
- 自动读取支付宝、微信或券商账户。
- 多人协作。
- 复杂记账分类。
- 税务计算。
- 专业投资分析。
